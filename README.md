为了方便用啊，方法肯定是留一个超级大统一的函数或库接口出去，
用的时候调这个函数就行了。
然后各种灵活性啊什么的，都是从这个函数的参数里传进去的。
这样的话，调用方只需要记住一个函数名就行了，
而且这个函数的参数设计得好，调用方也能很灵活地使用它。
另外，如果以后需要改动什么逻辑，只要改这个大函数的实现就行了，
调用方根本不用动，保持向后兼容性也更好。
所以说，留一个超级大统一的函数或库接口出去，
既方便调用方使用，又方便我们后续维护和扩展。
